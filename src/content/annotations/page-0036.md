Daniel's indexing system is described.

**"According to what scheme?"** — Daniel has organized his life's notes on cards using a system based on products of prime numbers. Each topic gets a prime; a card about multiple topics is tagged with their product. The number 30 (= 2 × 3 × 5) means a card sits at the intersection of three subjects. This is an indexing system that allows composite, multi-dimensional classification — no single linear ordering required.

**"It is the product of five primes"** — The wiki recognized this as a nod to Gödel numbering, where mathematical statements are encoded as products of primes. Given Daniel's connection to [Leibniz](/topic/gottfried-wilhelm-von-leibniz)'s computing ambitions, this is not a coincidence. Stephenson is building a lineage: Euclid → [Wilkins](/topic/john-wilkins)'s universal language → Leibniz → Daniel's Logic Mill → Gödel → Turing → the modern computer.

**Faceted classification** — One contributor identified Daniel's scheme as a precursor to Ranganathan's "colon classification" (1933), which lets you build composite categories from simpler ones. Libraries still use this principle. Daniel has independently invented what 20th-century information scientists would formalize — because the problem of organizing knowledge doesn't change, only the tools.
